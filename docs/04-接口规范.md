# 04-接口规范 — TradeX API 契约

> 本文档为 TradeX 完整的 API 接口规范，整合了 FSD.md §7、§8、§10、§11、§12、§13、§14、§17、§18、§19、§20、§21、§22 的全部内容。FSD.md 中标的相关章节由本文档替代。

---

## 1. 通用规范

### 1.1 统一响应体

```json
// 成功（单条）
{ "data": { ... } }

// 成功（列表）
{ "data": [ ... ], "total": 42 }

// 错误
{ "code": "ERROR_CODE", "message": "描述", "traceId": "00-xxx..." }
```

### 1.2 通用参数

| 参数 | 说明 |
|------|------|
| 认证头 | `Authorization: Bearer <accessToken>` |
| 翻页 | `page` (query, default 1), `pageSize` (query, default 20, max 100) |
| JSON 字段命名 | 统一小驼峰 (camelCase) |

### 1.3 HTTP 状态码约定

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 204 | 删除/操作成功（无响应体） |
| 400 | 请求参数非法 |
| 401 | 未认证或认证失败 |
| 403 | 权限不足或用户已禁用 |
| 404 | 资源不存在 |
| 409 | 冲突（状态冲突、重复、级联约束） |
| 500 | 内部错误 |
| 503 | 系统未初始化 |

## 2. API 端点清单

### 2.1 Setup

#### `GET /api/setup/status`
- Auth: **Public**
- 说明：查询系统初始化状态
- Response `200`:
```json
{
  "isInitialized": true
}
```

#### `POST /api/setup/initialize`
- Auth: **Public**（仅 Uninitialized 可用）
- 说明：执行系统初始化，创建 Super Admin（状态 `PendingMfa`），写入默认系统配置
- Request:
```json
{
  "userName": "superadmin",
  "password": "<secret>"
}
```
- Success: `204 No Content`
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `SETUP_ALREADY_INITIALIZED` | 409 | 系统已初始化 |
  | `SETUP_INVALID_INPUT` | 400 | 参数非法 |

### 2.2 Auth

#### `POST /api/auth/login`
- Auth: **Public**
- 说明：用户登录，返回 MFA 临时 token
- Request:
```json
{
  "userName": "string",
  "password": "string"
}
```
- Response `200`:
```json
{
  "mfaRequired": true,
  "mfaToken": "short-lived-token"
}
```
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `AUTH_INVALID_CREDENTIALS` | 401 | 用户名或密码错误 |
  | `AUTH_USER_DISABLED` | 403 | 用户已被禁用 |

#### `POST /api/auth/verify-mfa`
- Auth: **Public**
- 说明：验证 MFA TOTP 或恢复码，签发 JWT
- Request:
```json
{
  "mfaToken": "string",
  "totpCode": "123456",
  "recoveryCode": "xxxx-xxxx"
}
```
- `totpCode` 与 `recoveryCode` 二选一
- Response `200`:
```json
{
  "accessToken": "jwt-string",
  "refreshToken": "refresh-token-string",
  "expiresIn": 1800,
  "role": "Admin"
}
```
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `AUTH_MFA_INVALID_CODE` | 401 | MFA 验证码或恢复码错误 |

#### `POST /api/auth/refresh`
- Auth: **Public**
- 说明：刷新 AccessToken + RefreshToken（轮换策略，旧 RT 自动吊销）
- Request:
```json
{
  "refreshToken": "string"
}
```
- Response `200`:
```json
{
  "accessToken": "new-jwt-string",
  "refreshToken": "new-refresh-token-string",
  "expiresIn": 1800
}
```
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `AUTH_REFRESH_TOKEN_INVALID` | 401 | RefreshToken 无效或已吊销 |

#### `POST /api/auth/mfa/setup`
- Auth: **Bearer**（临时 token，login 返回）
- 说明：获取 MFA 绑定信息（密钥 + QR 码 + 恢复码）
- Request: 无请求体
- Response `200`:
```json
{
  "secretKey": "JBSWY3DPEHPK3PXP",
  "qrCodeUrl": "otpauth://totp/TradeX:user?secret=...",
  "qrCodeImage": "data:image/png;base64,...",
  "recoveryCodes": ["xxxx-xxxx", "..."]
}
```

#### `POST /api/auth/mfa/verify`
- Auth: **Bearer**（临时 token，login 返回）
- 说明：验证 MFA 绑定，完成激活
- Request:
```json
{
  "code": "123456"
}
```
- Response `200`:
```json
{
  "recoveryCodes": ["xxxx-xxxx", ...],
  "accessToken": "jwt-string",
  "refreshToken": "...",
  "expiresIn": 1800
}
```
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `AUTH_MFA_INVALID_CODE` | 400 | MFA 验证码错误 |

#### `POST /api/auth/logout`
- Auth: **Bearer**（已登录）
- 说明：吊销当前用户所有 RefreshToken
- Response: `200`
- 关联需求：FR-08.1

#### `POST /api/auth/send-recovery-codes`
- Auth: **Bearer**（Admin+）
- 说明：重新生成恢复码（旧码立即失效）
- Request:
```json
{
  "userId": "guid"
}
```
- Response: `200` + 新恢复码列表

### 2.3 Users

#### `GET /api/users`
- Role: **admin / super_admin**
- 说明：用户列表（翻页），不返回 PasswordHash / MfaSecret / RecoveryCodes
- Response `200`:
```json
{
  "data": [
    {
      "id": "guid",
      "userName": "string",
      "role": "Admin",
      "status": "Active",
      "createdAt": "2026-04-24T00:00:00Z"
    }
  ],
  "total": 10
}
```

#### `POST /api/users`
- Role: **admin / super_admin**
- 说明：创建用户（状态为 `PendingMfa`）
- Request:
```json
{
  "userName": "string",
  "password": "string",
  "role": "Operator"
}
```
- Response: `201` + User 对象（不含敏感字段）
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `AUTH_USERNAME_EXISTS` | 409 | 用户名已存在 |
  | `AUTH_INVALID_ROLE` | 400 | 不可创建 SuperAdmin；Admin 不可创建/分配 Admin 角色 |

#### `PUT /api/users/:id/role`
- Role: **super_admin**（仅 SuperAdmin 可变更）
- 说明：变更用户角色
- Request:
```json
{
  "role": "Operator"
}
```
- Response: `200`
- 约束：Super Admin 不可降级
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `USER_CANNOT_DOWNGRADE_SUPER_ADMIN` | 400 | 不可降级 Super Admin |

#### `PUT /api/users/:id/status`
- Role: **admin / super_admin**
- 说明：启用/禁用用户
- Request:
```json
{
  "status": "Disabled"
}
```
- Response: `200`
- 约束：Super Admin 不可禁用
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `USER_CANNOT_DISABLE_SUPER_ADMIN` | 400 | 不可禁用 Super Admin |

  | `TRADER_HAS_ACTIVE_STRATEGIES` | 409 | 有关联活跃策略，需先停用 |

```json
{
```

### 2.6 Strategies（策略模板）

策略模板是纯条件树 + 执行规则的复用单元，**不绑定**交易所、交易对或时间周期。

#### `GET /api/strategies`
- Role: **all authenticated**
- 说明：获取策略模板列表
- Response `200`:
```json
{
  "data": [
    {
      "id": "guid",
      "name": "RSI Strategy",
      "entryConditionJson": "...",
      "exitConditionJson": "...",
      "executionRuleJson": "...",
      "version": 3,
      "createdAt": "2026-04-24T00:00:00Z"
    }
  ]
}
```

#### `POST /api/strategies`
- Role: **admin / operator**
- 说明：创建策略模板
- Request:
```json
{
  "name": "RSI Strategy",
  "entryConditionJson": "{ ... }",
  "exitConditionJson": "{ ... }",
  "executionRuleJson": "{ ... }"
}
```
- Response: `201` + Strategy 对象

#### `GET /api/strategies/:id`
- Role: **all authenticated**
- 说明：获取策略模板详情
- Response: 完整策略模板对象

#### `PUT /api/strategies/:id`
- Role: **admin / operator**
- 说明：更新策略模板
- Request: 同 POST，所有字段可选
- Response: `200`
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `STRATEGY_NOT_FOUND` | 404 | 策略不存在 |

#### `DELETE /api/strategies/:id`
- Role: **admin / operator**
- Response: `204`

### 2.7 Strategy Deployments（策略部署）

策略部署是将策略模板绑定到某 Trader 下指定作用域的操作。作用域自动推导：
- **Pair 作用域**：`Pairs` 不为空
- **Exchange 作用域**：`Pairs` 为空且 `ExchangeId` 不为空
- **Trader 作用域**：`Pairs` 为空且 `ExchangeId` 为空

#### `GET /api/traders/{traderId}/strategies`
- Role: **all authenticated**
- 说明：获取交易员下的策略部署列表
- Response `200`:
```json
{
  "data": [
    {
      "id": "guid",
      "strategyId": "guid",
      "traderId": "guid",
      "exchangeId": "guid",
      "Pairs": "[\"guid1\"]",
      "timeframe": "15m",
      "status": "Active",
      "scope": "Pair",
      "createdAt": "2026-04-24T00:00:00Z"
    }
  ]
}
```

#### `POST /api/traders/{traderId}/strategies`
- Role: **admin / operator**
- 说明：创建策略部署（状态为 `Draft`）
- Request:
```json
{
  "strategyId": "guid",
  "exchangeId": "guid",
  "Pairs": "[\"guid1\", \"guid2\"]",
  "timeframe": "15m"
}
```
- Response: `201`
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `STRATEGY_KLINE_WARMUP_FAILED` | 400 | K 线预热失败 |

#### `PUT /api/traders/{traderId}/strategies/{id}`
- Role: **admin / operator**
- 说明：更新策略部署
- 约束：`Active` 状态不可编辑，需先停用
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `STRATEGY_NOT_EDITABLE` | 400 | 当前状态不可编辑 |

#### `DELETE /api/traders/{traderId}/strategies/{id}`
- Role: **admin / operator**
- 说明：删除策略部署（级联删除关联回测任务和结果）
- 约束：`Active` 不可删除
- Response: `204`
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `STRATEGY_IS_ACTIVE` | 409 | 活跃策略不可删除 |

#### `POST /api/traders/{traderId}/strategies/{id}/toggle`
- Role: **admin / operator**
- 说明：启用/停用策略部署
- Request:
```json
{
  "enable": true
}
```
- `enable: true`：约束部署状态必须为 `Passed`，且无 Pair 级冲突
- `enable: false`：约束部署状态必须为 `Active`
- Error:
  | Code | HTTP | 说明 |
  |------|------|------|
  | `STRATEGY_CANNOT_ENABLE` | 400 | Draft 状态不可启用 |
  | `STRATEGY_EXCHANGE_CONFLICT` | 409 | 同一 Trader 在同一 Exchange 上对同一 Pair 已有活跃策略 |

### 2.10 Backtests

#### `GET /api/traders/{traderId}/strategies/{strategyId}/backtests/tasks`
- Role: **all authenticated**
- 说明：按 StrategyId 查询回测任务列表
- Response: `200` + `BacktestTask[]`

#### `POST /api/traders/{traderId}/strategies/{strategyId}/backtests`

```csharp
public sealed class TradingHubClient : IAsyncDisposable
{
    private HubConnection _hubConnection;

    public async Task InitializeAsync(string hubUrl, string accessToken)
    {
        _hubConnection = new HubConnectionBuilder()
            .WithUrl(hubUrl, options => options.AccessTokenProvider = () => Task.FromResult(accessToken))
            .WithAutomaticReconnect()
            .Build();

        _hubConnection.On<Position>("PositionUpdated", position => OnPositionUpdated?.Invoke(position));
        _hubConnection.On<Order>("OrderPlaced", order => OnOrderPlaced?.Invoke(order));
        _hubConnection.On<StrategyStatus>("StrategyStatusChanged", status => OnStrategyStatusChanged?.Invoke(status));
        _hubConnection.On<RiskAlert>("RiskAlert", alert => OnRiskAlert?.Invoke(alert));
        _hubConnection.On<DashboardSummary>("DashboardSummary", summary => OnDashboardSummary?.Invoke(summary));
        _hubConnection.On<ExchangeConnectionStatus>("ExchangeConnectionChanged", status => OnExchangeConnectionChanged?.Invoke(status));

        await _hubConnection.StartAsync();
    }

    public event Action<Position>? OnPositionUpdated;
    public event Action<Order>? OnOrderPlaced;
    public event Action<StrategyStatus>? OnStrategyStatusChanged;
    public event Action<RiskAlert>? OnRiskAlert;
    public event Action<DashboardSummary>? OnDashboardSummary;
    public event Action<ExchangeConnectionStatus>? OnExchangeConnectionChanged;

    public async ValueTask DisposeAsync()
    {
        await _hubConnection.DisposeAsync();
    }
}
```

## 4. 条件树引擎规格

### 4.1 数据结构

条件树是嵌套 JSON 结构，支持任意深度嵌套：

```json
{
  "operator": "AND",
  "conditions": [
    {
      "operator": "OR",
      "conditions": [
        {
          "indicator": "RSI",
          "parameters": { "period": 14 },
          "comparison": ">",
          "value": 70
        },
        {
          "indicator": "RSI",
          "parameters": { "period": 14 },
          "comparison": "<",
          "value": 30
        }
      ]
    },
    {
      "operator": "NOT",
      "conditions": [
        {
          "indicator": "SMA",
          "parameters": { "period": 50, "source": "close" },
          "comparison": ">",
          "value": 50000
        }
      ]
    }
  ]
}
```

### 4.2 节点类型

| 节点类型 | 子节点 | 说明 |
|----------|--------|------|
| `AND` | conditions[] | 全部条件同时满足 |
| `OR` | conditions[] | 至少一个条件满足 |
| `NOT` | conditions[0] | 条件取反（仅一个子节点） |
| 叶节点 | 无 | `indicator` + `parameters` + `comparison` + `value` |

### 4.3 叶节点字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `indicator` | string | 指标名称，首批支持：RSI / MACD / SMA / EMA / BB / VolumeSMA / OBV / KDJ |
| `parameters` | object | 指标参数键值对，如 `{ "period": 14 }` |
| `comparison` | enum | `>` / `<` / `>=` / `<=` / `==` / `CrossAbove` / `CrossBelow` |
| `value` | number | 阈值或基准值 |

### 4.4 约束规则

| 约束 | 值 |
|------|-----|
| 最大嵌套深度 | ≤ 5 层 |
| NOT 子节点数 | 仅 1 个 |
| Indicator 节点 | 必须为叶节点 |
| 首批指标 | RSI / MACD / SMA / EMA / BB / VolumeSMA / OBV / KDJ |

### 4.5 条件评估合约

```csharp
public interface IConditionTreeEvaluator
{
    /// <summary>评估单个条件节点是否满足</summary>
    bool Evaluate(ConditionNode node, List<Candle> candles);
```

### 4.6 Volatility Grid 上下文变量

| 名称 | 类型 | 说明 |
|------|------|------|
| `RANGE_PCT` | decimal | 当前 K 线波幅百分比，公式 `(high - low) / open * 100` |
| `AVG_ENTRY_PRICE` | decimal | 当前持仓均价 |

### 5.7 多层级组合风控

```csharp
public interface IPortfolioRiskManager
{
    bool CheckSystemLevel();
    bool CheckTraderLevel(Guid traderId);
    bool CheckExchangeLevel(Guid exchangeId);
    bool CheckPairLevel(string pair);
}
```

风控管线顺序：
```
入市请求
  → [1] 滑点检查 (SlippageCheck)
  → [2] 日亏损检查 (DailyLossCheck)
  → [3] 最大回撤检查 (MaxDrawdownCheck)
  → [4] 连续亏损检查 (ConsecutiveLossCheck)
  → [5] 交易频率熔断 (FreqCircuitBreaker)
  → [6] 冷却期检查 (CooldownCheck)
  → [7] 最大持仓检查 (MaxPositionCheck)
  → [通过] 执行下单
```

各检查项：
| 检查项 | 触发条件 | 处理方式 |
|--------|----------|----------|
| 滑点检查 | 预估滑点 > `slippageTolerancePercent` | 拒绝并记录 |
| 日亏损检查 | 当日累计已实现亏损 > `maxDailyLossPercent` | 暂停全账户，次日 00:00 UTC 恢复 |
| 最大回撤检查 | 账户净值回撤 > `maxDrawdownPercent` | 暂停全账户，通知管理员 |
| 连续亏损检查 | 策略连续亏损 ≥ `consecutiveLossLimit` | 暂停该策略，通知 |
| 交易频率熔断 | 单策略 5 分钟内触发买入 ≥ 3 次 | 跳过本次，冷却延长 |
| 冷却期检查 | 距上次交易 < `cooldownSeconds` | 跳过本次 |
| 最大持仓检查 | 当前持仓数 ≥ `maxPositionCount` | 跳过买入 |

### 7.3 初始化模式下的特例

SetupGuard 中间件直接短路：非 Setup/Health 请求一律返回 `503 SYSTEM_NOT_INITIALIZED`，避免"未初始化但 JWT 可通过"的歧义路径。

### 7.4 中间件流程

1. 请求到达 → SetupGuard 检查初始化状态
2. 未初始化 → 仅放行 `/health` + `/api/setup/*`
3. 已初始化 → JWT 中间件验签 → 取 `sub(role)` + `sub(userId)`
4. Casbin `Enforce(role, path, method)` → false → 403

Casbin 不参与 MFA 验证。MFA 在 JWT 签发前完成，是登录的前置条件。

## 9. 安全规格

| 安全项 | 规格 |
|--------|------|
| API Key / Secret / Passphrase | AES-256-GCM 加密后入库，密钥来源 `jwt.secret`（派生）或独立密钥 |
| IP 白名单 | 可选开关（`Security.IpWhitelist.Enabled`），开启后拦截非白名单 IP 请求；白名单列表支持 CIDR 格式；API Key 测试连接时校验交易所后台 IP 白名单配置，若未配置则给出警告但不拒绝 |
| 密码 | bcrypt 哈希，cost 默认 12，可配置 |
| MFA Secret | AES-256-GCM 加密存储 |
| Recovery Codes | 只存 bcrypt 哈希，初始化生成 8 个，单次使用 |
| JWT AccessToken | 默认 30 分钟，`sub=userId`，`role` 放入 claims |
| JWT RefreshToken | 默认 7 天，随机字符串 + 数据库持久化（可吊销），轮换策略（每次使用刷新） |
| RefreshToken 轮换 | 每次使用 RefreshToken 时签发新 RT + 吊销旧 RT（旧 RT 放入黑名单到过期） |
| MFA Token | 短生命周期（5 分钟），用于 login → verify-mfa 中间状态 |
| Casbin | 模型配置文件 + CSV 策略文件存入 `TradeX.Infrastructure`，应用启动时加载 |
| HTTPS | 生产环境通过反向代理（Nginx）终止 TLS |
| CORS | 仅允许前端域名，开发环境放宽 |
| Swagger | 仅 Development 环境暴露 |
| 审计日志 | 敏感写操作自动记录 |

### 10.3 审计日志自动解析规则

AuditLogMiddleware 路径解析规则：
- 路径解析为中文可读名称：`POST /api/traders` → 操作="新建交易员"，资源="交易员"
- 已知资源名映射：`traders`→交易员, `exchanges`→交易所, `strategies`→策略, `users`→用户, `auth`→认证, `orders`→订单, `settings`→系统设置, `notifications`→通知渠道, `positions`→持仓
- 已知子操作：`toggle`→启用/禁用, `test`→测试连接, `backtests`→回测
- Detail 字段存储技术信息 JSON：`{ method, path, statusCode, ip }`

### 10.4 审计日志存储

- 表：`AuditLog`（SQLite）
- 定期清理策略：保留最近 6 个月，超期归档为压缩 JSON 文件
