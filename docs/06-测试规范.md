## 4. 测试方法与规范

### 4.1 单元测试
- 遵循 FIRST 原则：Fast / Independent / Repeatable / Self-Validating / Timely
- 使用 NSubstitute Mock 外部依赖（交易所 SDK、数据库）
- 覆盖正常路径 + 边界条件 + 异常路径
- 测试数据构造使用工厂方法，禁止硬编码

### 4.2 集成测试
- API 集成测试通过 `WebApplicationFactory<T>` 启动测试宿主
- 使用内存 SQLite 或测试数据库文件
- 覆盖完整的请求-响应周期（含中间件管线）

### 4.3 E2E 测试
- Playwright 浏览器自动化
- 覆盖关键用户旅程：初始化→登录→创建策略→回测→交易
- 运行在预发布环境 (Staging)

### 7.1 M1 — Volatility Grid 策略专项

#### TC-STRAT-VG-001: 波幅触发首单
- **前置条件**: `volatility_grid`，`entryVolatilityPercent=1.0`，无持仓
- **测试步骤**: 注入 `RANGE_PCT=1.2` 的 K 线，执行策略评估
- **预期结果**: 触发首单买入
- **关联需求**: FR-03.11

#### TC-STRAT-VG-002: 波幅不足不触发首单
- **前置条件**: 同上
- **测试步骤**: 注入 `RANGE_PCT=0.8` 的 K 线，执行评估
- **预期结果**: 不下单

#### TC-STRAT-VG-003: 价格低于均价 1% 触发加仓
- **前置条件**: 持仓均价 100
- **测试步骤**: 当前价 99，执行评估
- **预期结果**: 加仓，`pyramidingCount+1`

#### TC-STRAT-VG-004: 价格高于均价 1% 触发减仓
- **前置条件**: 持仓均价 100
- **测试步骤**: 当前价 101，执行评估
- **预期结果**: 减仓

#### TC-STRAT-VG-005: 最大追加次数后禁止加仓
- **前置条件**: `maxPyramidingLevels=5`，当前 `pyramidingCount=5`
- **测试步骤**: 满足加仓条件
- **预期结果**: 不加仓

#### TC-STRAT-VG-006: noStopLoss=true 不触发仓位止损
- **前置条件**: `noStopLoss=true`
- **测试步骤**: 价格快速下跌，执行评估
- **预期结果**: 仓位止损分支不执行

#### TC-STRAT-VG-007: noStopLoss=true 仍受账户级风控
- **前置条件**: `noStopLoss=true`，日亏超限
- **测试步骤**: 继续触发买入
- **预期结果**: 风控拒绝

#### TC-STRAT-VG-008: 双周期部署去重
- **前置条件**: 同一策略部署 5m 与 15m
- **测试步骤**: 去重窗口内同时触发
- **预期结果**: 仅一笔订单

#### TC-STRAT-VG-009: 回测与实盘决策一致
- **前置条件**: 同一组历史 K 线
- **测试步骤**: 执行回测和实盘模拟
- **预期结果**: 触发点、加减仓一致

#### TC-STRAT-VG-010: 参数校验
- **前置条件**: `rebalancePercent <= 0`
- **测试步骤**: 提交策略
- **预期结果**: 参数校验错误
