## 2. C# 14 强制语法

所有代码必须使用以下 C# 14 语法，禁止使用旧式写法。

### 2.1 主构造函数

消除所有简单 DI 字段注入：

```csharp
// ✅ 正确：主构造函数
public class TradingService(ITradeExecutor executor, ILogger<TradingService> logger)
    : ITradingService
{
}

// ❌ 错误：传统 DI 字段注入
public class TradingService : ITradingService
{
    private readonly ITradeExecutor _executor; // 禁止
    private readonly ILogger<TradingService> _logger; // 禁止

    public TradingService(ITradeExecutor executor, ILogger<TradingService> logger)
    {
        _executor = executor;
        _logger = logger;
    }
}
```

**例外**：`IOptions<T>.Value` 延迟访问、循环依赖场景仍可使用传统字段注入，但必须加注释说明理由。

```csharp
// ✅ 例外：延迟访问 IOptions<T>.Value
public class TradingService(IOptionsMonitor<JwtSettings> jwtOptions)
{
    // IOptions<T>.Value 在构造函数中不可用，保留字段
    private string? _secret;

    public void Init()
    {
        _secret = jwtOptions.CurrentValue.Secret;
    }
}
```

### 2.2 集合表达式

```csharp
// ✅ 正确
[]                              // 空列表/空数组
[item1, item2]                  // 列表/数组初始化
[.. existingList]               // 展开运算符
Dictionary<string, int> { ["key"] = 1 }  // 字典初始化

// ❌ 禁止
new List<T>()                    // 用 []
Array.Empty<T>()                 // 用 []
new Dictionary<K,V>()            // 用 []
new List<T> { item1, item2 }     // 用 [item1, item2]
```

### 2.3 `field` 关键字

```csharp
// ✅ 正确
private string _name => field;

// ❌ 禁止：手动声明 backing field
private string _name;
public string Name
{
    get => _name;
    set => _name = value;
}
```

## 5. 编码约定

### 5.1 通用

- **Nullable 启用**：所有引用类型必须标注可为空（`string?` vs `string`），`<Nullable>enable</Nullable>`
- **CancellationToken**：所有 I/O 方法必须传递 `CancellationToken` 参数，默认值 `default`
- **Async 后缀**：所有异步方法名必须以 `Async` 结尾
- **结构化日志**：使用 Serilog 结构化占位符，禁止字符串拼接
- **无 Service Locator**：禁止 `IServiceProvider.GetService()`（仅 `Program.cs` 允许）
- **输入验证**：所有外部输入（请求参数、查询参数、请求体）视为不可信，必须在边界处验证
- **强类型配置**：所有可调参数使用 `IOptions<T>` / `IOptionsMonitor<T>` 绑定，禁止硬编码魔力值

CancellationToken 传递：

```csharp
// ✅ 正确：传递 CancellationToken
public async Task<OrderResult> PlaceOrderAsync(OrderRequest request, CancellationToken ct)
{
    return await _exchangeClient.PlaceOrderAsync(request, ct);
}

// ❌ 错误：省略 CancellationToken
public async Task<OrderResult> PlaceOrderAsync(OrderRequest request)
{
    return await _exchangeClient.PlaceOrderAsync(request, CancellationToken.None); // 禁止
}
```

强类型配置：

```csharp
// ✅ 正确：强类型 Options
public class JwtSettings
{
    public string Secret { get; init; }
    public int AccessTokenExpiresMinutes { get; init; } = 30;
}

// ❌ 禁止：字符串散值
int expiresIn = 30;  // 禁止
```

### 5.2 结构化日志

```csharp
// ✅ 正确：结构化日志，使用占位符
logger.LogInformation("Order {OrderId} placed for {Pair} at {Price}", order.Id, pair, price);

// ❌ 禁止：字符串拼接
logger.LogInformation($"Order {order.Id} placed for {pair}");  // 禁止

// ✅ 正确：异常日志使用异常重载
logger.LogError(ex, "Failed to place order {OrderId}", orderId);

// ❌ 禁止：异常作为消息字符串拼接
logger.LogError($"Failed to place order {orderId}: {ex.Message}");  // 禁止
```

## 6. 错误处理策略

| 层级 | 策略 | 说明 |
|------|------|------|
| Controller | 全局异常过滤器 `ExceptionHandlingMiddleware` | 500 → 统一 JSON 错误响应（含 `traceId`）+ 日志 |
| Service / Business | 抛出 `DomainException` | 400/404/409 → 带业务错误码 |
| Trading Engine | catch + log + 继续循环 | 单策略异常不影响其他策略；连续 N 次异常暂停该策略 |
| Exchange Client | catch API 异常 → 转换为 `ExchangeException` | 包含错误码、HTTP 状态、原始消息 |
| WebSocket | 断线不放 crash | 指数退避重连（1s → 2s → 4s → ... → 30s max），超上限后标记 `Disconnected` |

统一错误响应体：

```json
{
  "code": "AUTH_INVALID_CREDENTIALS",
  "message": "Invalid username or password.",
  "traceId": "00-abc123..."
}
```

## 8. 并发策略

| 场景 | 策略 | 说明 |
|------|------|------|
| 同一 Pair 同时触发买入 | 策略级锁 + 风控熔断 | Trading Engine 同周期内同一 Pair 仅执行一次 |
| 手动下单 vs 策略下单 | 无冲突（共用风控链） | 手动下单同样经过滑点 + 日亏损检查 |
| 策略编辑 vs 策略评估 | 乐观并发（Version 字段） | `UPDATE ... WHERE Version = @expected` |
| 订单状态更新 | 幂等更新 | `Order.Status = Filled` 可重复执行，不报错 |
| 回测多任务同时执行 | 资源感知调度 | Worker 池固定 `MaxConcurrency` 个，`ResourceMonitor` 每 5s 采样内存 + CPU，`AllowedConcurrency = min(mem_cap, cpu_cap)`，按双维度水位动态限制活跃并发数；超过水位的新任务等待槽位释放，不中断进行中的任务 |
| WebSocket 数据写入 | 内存缓存 + 时序数据库 | 接收数据后写入 `ConcurrentDictionary` 内存缓存 + K 线走时序库 |

## 9. 安全边界

| 安全域 | 信任边界 | 控制点 |
|--------|---------|--------|
| 浏览器 ↔ Nginx | 不可信 | HTTPS（生产）、CORS、XSS 防护（前端输出编码） |
| Nginx ↔ Backend | 可信内网 | Docker 内网通信 |
| Backend ↔ 交易所 | 不可信 | API Key 仅内存解密；错误处理不泄露凭证 |
| Backend ↔ SQLite | 可信 | 文件权限 0600 |

**安全实现要点**：
- JWT AccessToken（默认 30 分钟）+ RefreshToken（默认 7 天，数据库持久化，可吊销，轮换策略）
- MFA TOTP 强制绑定，MFA Token 有效期 5 分钟
- Casbin RBAC（`keyMatch3 + regexMatch`），策略与代码分离（model.conf + policy.csv）
- AES-256-GCM 加密 API Key / MFA Secret / Passphrase
- bcrypt 哈希密码 + Recovery Codes（只存哈希，单次使用）
- 敏感操作自动记录 AuditLog
- 审计日志忽略 GET/HEAD/OPTIONS 无变更操作

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| Trading Engine 评估周期 | ≤ 15s | 日志记录 cycle start/end |
| WebSocket 数据延迟 | ≤ 2s（交易所 → 内存缓存） | 时间戳比对 |
| REST API P95 响应时间 | ≤ 500ms | 中间件记录 |
| SignalR 推送延迟 | ≤ 1s | 客户端时间戳 |
| 回测 30 天 15m K 线 | ≤ 10s | 计时 |

## 12. 后端禁止事项

- ❌ 禁止擅自修改接口契约（签名、返回格式、错误码）
- ❌ 禁止绕过 PM 和架构师引入新的第三方依赖
- ❌ 禁止直接操作生产数据库（只能通过迁移脚本变更 Schema）
- ❌ 禁止将敏感信息（密钥、连接串、用户隐私）硬编码或提交到代码仓库
- ❌ 禁止跳过 QA 环节直接部署或发布
- ❌ 禁止在前端不知情的情况下修改 API 响应格式

## 13. 前端禁止事项

- ❌ 禁止绕过 Blazor 声明式机制直接操作 DOM（除非必要性能优化且有注释说明）
- ❌ 禁止在前端代码中硬编码后端 API 地址（必须从环境变量或配置读取）
- ❌ 禁止将敏感 Token/密钥存储在前端本地存储明文
- ❌ 禁止擅自修改或新增后端 API 接口
- ❌ 禁止跳过类型检查/静态分析提交代码
- ❌ 禁止忽略 Lint 工具报告的任何 Error 级别问题
- ❌ 禁止绕过 UI/UX 设计稿自行决定关键交互布局

## 14. 前端编码约束

### 14.1 组件库与框架

| 约束项 | 要求 |
|--------|------|
| 框架 | Blazor Server (InteractiveServer)，.NET 10 Interactive Server 渲染模式 |
| 组件库 | `BootstrapBlazor` v10.6.0，Bootstrap 5 设计系统 |
| 状态管理 | DI Singleton 服务 + `@implements IAsyncDisposable` 生命周期管理，按业务域拆分服务 |
| HTTP 客户端 | `HttpClient` + `IHttpClientFactory`，内置 `Authorization` 消息处理器自动附加 JWT |
| 实时通信 | `HubConnection` + `@implements IAsyncDisposable`，强类型 C# 订阅模式 |
| 图表 | ECharts + `ECharts` Blazor 封装 |

### 14.2 页面分类与路由

- 页面按业务域拆分为独立目录：`Exchanges/`、`Strategies/`、`Backtests/`、`Positions/`、`Orders/` 等
- 所有 Razor 组件位于 `TradeX.Api/Components/` 下
- 页面路由通过 `@page` 指令声明
- 角色守卫使用 `AuthorizeView` + `[Authorize(Roles=)]` 两级控制
- 前端角色策略与后端 Casbin RBAC 策略保持一致

### 14.3 共享组件

公共 UI 组件抽离至 `Components/Shared/` 目录，包括：
- 条件树编辑器（ConditionTreeEditor）
- 执行规则表单（ExecutionRuleEditor）
- 通用表格、通用弹窗等

### 14.4 前端服务拆分

按业务域拆分为独立服务（Auth、Trader、Exchange、Strategy、Position、Order、Dashboard 等），通过 DI 注入。服务通过 SignalR 事件总线接收后端推送。
